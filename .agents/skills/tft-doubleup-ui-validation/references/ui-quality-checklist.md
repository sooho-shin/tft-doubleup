# UI 품질 체크리스트

## Figma 산출물 구조

- 필요한 Figma 페이지가 모두 존재하는가
- 각 페이지 children 수가 0이 아닌가
- `05 Pages`에 최종 화면 프레임이 모두 존재하는가
- 최종 화면 프레임 크기가 명세와 맞는가
- 스크린샷을 찍은 node id가 실제 최종 프레임인가
- Figma MCP 조회가 빈 children을 반환하면 같은 파일을 재조회하거나 스크린샷으로 실제 존재 여부를 확인했는가

기본 페이지 세트:

- `00 Foundation`
- `01 Atoms`
- `02 Molecules`
- `03 Organisms`
- `04 Templates`
- `05 Pages`
- `99 Notes`

기본 최종 프레임:

- `Page/Desktop Matching Board`
- `Page/Desktop Recruitment Detail`
- `Page/Mobile Matching Board`

## 레이아웃

- 화면 전체가 명확한 부모-자식 구조인가
- 주요 섹션이 오토레이아웃/flexbox 구조로 설명 가능한가
- 섹션 간 기준선이 자연스럽게 맞는가
- 헤더, 검색, 콘텐츠 그리드, 사이드 패널의 폭과 위치가 균형적인가
- 한쪽으로 쏠린 것처럼 보이는 영역이 없는가
- 불필요한 absolute/좌표 배치가 없는가
- 반복 리스트 아이템이 배지와 텍스트를 부모 프레임으로 묶고 있는가
- 우측 패널과 모바일 카드 내부가 좌표 나열이 아니라 행 단위 구조로 설명 가능한가
- 왼쪽 필터 패널의 필터 그룹과 옵션 칩이 부모 패널 폭 안에서 wrap 또는 fill container로 정리되는가
- 주요 컨테이너의 자식 요소가 부모 `absoluteBoundingBox` 밖으로 나가지 않는가

## Spacing / Padding

- 버튼의 좌우 padding이 텍스트 길이에 비해 자연스러운가
- 버튼의 상하 padding이 높이와 글자 크기에 맞는가
- 입력 필드, 셀렉트, 토글의 높이가 서로 어울리는가
- 카드 내부 padding이 너무 좁거나 넓지 않은가
- 카드 간 gap이 일정한가
- 반복 모집 카드의 카드 간 gap이 모두 같은가
- 마지막 모집 카드와 viewport/부모 섹션 하단 사이에 충분한 여백이 있는가
- 섹션 간 간격이 정보 위계와 맞는가
- 숫자 배지와 텍스트 그룹 사이 gap이 최소 8px 이상으로 보이는가
- 작은 패널 리스트의 행 간격이 너무 좁아 텍스트가 서로 붙어 보이지 않는가

일반 기준:

- 작은 태그: 높이 22-28px 수준
- 일반 버튼: 높이 32-40px 수준
- 주요 액션 버튼: 높이 40-48px 수준
- 카드 내부 padding: 12-20px 수준
- 촘촘한 리스트 gap: 8-12px 수준
- 주요 섹션 gap: 16-24px 수준

숫자는 절대값이 아니라 이상치 탐지 기준이다. 화면 밀도와 목적에 맞으면 조정 가능하다.

## 정렬

- 헤더 로고, 네비, 액션이 같은 기준선에 있는가
- 카드 제목, 메타 정보, 태그, 액션의 정렬이 일관적인가
- 모든 반복 모집 카드의 `신청하기` 버튼 right edge가 같은 x 좌표에 있는가
- 모든 반복 모집 카드의 `신청하기` 버튼 top offset이 카드 top 기준으로 같은가
- 모든 반복 모집 카드의 `Actions` row top offset이 카드 top 기준으로 같은가
- 작성자 프로필 행과 `신청하기` 버튼이 같은 행에서 수직 중앙 정렬되어 있는가
- 왼쪽 필터와 중앙 리스트의 상단이 맞는가
- 오른쪽 패널의 제목과 항목 정렬이 일정한가
- 배지와 텍스트의 수직 중앙 정렬이 어색하지 않은가
- 숫자 배지가 제목 텍스트를 침범하지 않는가
- 최근 모집글/내 신청 상태 패널의 모든 행에서 배지, 제목, 메타 텍스트의 시작점이 일관적인가
- 왼쪽 필터 패널의 모든 필터 그룹 시작점과 옵션 칩 row가 같은 기준선에 있는가
- 필터 옵션 칩이 중앙 모집 리스트 카드 영역을 침범하지 않는가

## 타이포그래피

- 페이지 제목, 섹션 제목, 카드 제목, 본문, 메타 텍스트의 크기 위계가 명확한가
- 작은 텍스트가 다크 배경에서 읽히는가
- 제목이 카드 안에서 과하게 크지 않은가
- 버튼 텍스트가 버튼 안에서 중앙 정렬되어 있는가
- 줄바꿈이 의미 단위와 맞는가
- 한국어 배지/태그 라벨이 의도 없이 두 줄로 쪼개지지 않는가
- 작은 메타 텍스트가 너무 어두워 실제로 읽을 수 없는 수준은 아닌가

## 컴포넌트

- 버튼 상태가 명확한가
- disabled/secondary/primary의 차이가 과하거나 부족하지 않은가
- 태그와 배지가 버튼처럼 오해되지 않는가
- 입력 필드가 클릭 가능한 요소처럼 보이는가
- 모집 카드가 반복되어도 구조가 흔들리지 않는가
- 반복 모집 카드의 `Header`, `Title`, `Meta`, `Tags`, `Actions` 행이 카드마다 같은 상대 좌표 리듬을 유지하는가
- 컴포넌트 이름이 Atomic Design 계층에 맞는가
- `RankBadge + TextGroup` 형태의 리스트 행이 컴포넌트 또는 프레임 단위로 재사용 가능한가
- 카드의 `Header`, `Meta`, `Tags`, `Actions` 행이 서로 겹치지 않는가
- 모바일 카드의 신청 버튼이 카드 밖으로 밀리지 않는가

## 정보 위계

- KR 서버, 티어, 지금 가능 여부가 빠르게 보이는가
- 제목보다 부가 태그가 더 튀지 않는가
- 신청 버튼이 보이지만 정보보다 과하게 지배하지 않는가
- 최근 모집글/내 신청 상태/공지 패널이 메인 모집 리스트를 방해하지 않는가
- MVP에서 실제 파트너 추천/랭킹/가이드 패널이 노출되지 않는가

## 반응형

- 데스크톱에서 콘텐츠가 과하게 넓게 퍼지지 않는가
- 모바일에서 카드가 단일 컬럼으로 자연스럽게 쌓이는가
- 모바일 터치 영역이 충분한가
- 긴 닉네임, 긴 태그, 긴 제목이 UI를 깨지 않는가
- 필터는 모바일에서 바텀시트나 접힘 구조로 전환 가능한가
- 모바일 하단 액션바가 viewport 밖으로 내려가지 않는가
- 모바일 카드의 즉시 가능 상태/디스코드/목적 배지가 오른쪽 경계를 넘지 않는가
- 데스크톱 필터 패널의 긴 한국어 옵션 칩이 부모 패널 밖으로 밀리지 않는가

- 상세 화면의 `비슷한 모집글`이 확대 상태에서도 깨지지 않는가

- 패널 리스트 행의 제목과 메타가 서로 겹치지 않는가

## 모든 요소 전수 검증

전체 화면 단위 검증만으로는 불충분하다. 모든 최종 화면의 모든 요소를 하나씩 검사해야 한다.

공통 전수 검증 항목:

- 요소가 부모 영역 밖으로 넘치지 않는가
- 요소가 다른 요소와 겹치지 않는가
- 텍스트가 의도 없이 줄바꿈되지 않는가
- 텍스트가 잘리지 않는가
- 텍스트가 너무 작거나 흐려 읽기 어려운 수준은 아닌가
- 배지/태그/버튼 안의 텍스트가 수직/수평 중앙에 있는가
- 배지/태그/버튼 내부 padding이 충분한가
- 인접 요소와 최소 8px 이상의 시각적 gap이 있는가
- 반복 요소의 x/y 시작점과 기준선이 일관적인가
- 요소 크기가 콘텐츠 길이에 비해 너무 작거나 크지 않은가

왼쪽 필터 패널 필수 검증:

- `Organism/Filters/Sidebar` 컨테이너
- 모든 `Molecule/FilterGroup/*`
- 각 필터 그룹 제목
- 각 필터 그룹의 모든 옵션 칩
- `필터 적용` 버튼
- `초기화` 버튼
- 필터 패널과 중앙 모집 리스트 사이의 경계
- 필터 그룹 또는 옵션 칩이 `Organism/Filters/Sidebar` 좌우 경계를 1px 이상 넘지 않는지
- 옵션 칩 row가 부모 폭보다 넓은 hug contents 상태로 남아 있지 않은지
- `clipsContent`로 overflow를 숨기는 방식이 아닌지

## 부모 영역 overflow 기하 검증

스크린샷 검증과 별도로 Figma 노드의 `absoluteBoundingBox`를 확인한다. 하나라도 부모 경계를 벗어나면 `불합격`이다.

검사 대상:

- `Organism/Filters/Sidebar`
- 모든 `Molecule/FilterGroup/*`
- `Organism/RecruitmentCard/*`
- `Layout/RightRail`
- 모든 `Organism/Panel/*`
- `Page/Mobile Matching Board` 내부 카드와 하단 액션바

불합격 조건:

- 자식의 right edge가 부모 right edge를 1px 이상 넘는다.
- 자식의 left edge가 부모 left edge를 1px 이상 넘는다.
- 자식의 bottom edge가 부모 bottom edge를 1px 이상 넘고 의도된 스크롤/클리핑 구조가 아니다.
- 텍스트, 배지, 태그, 버튼이 부모 컨테이너 밖으로 나간다.
- 허용된 overlay가 아닌데 부모 밖에 위치한다.
- `clipsContent=true`로 overflow를 숨겨 통과처럼 보이게 만든다.

허용 예외:

- `준비중입니다.` 모달, 드롭다운, 알림 배지처럼 의도된 overlay이며 레이어 이름과 목적이 명확한 경우
- 최종 화면 viewport 밖으로 나가지 않고 다른 정보 영역을 침범하지 않는 팝오버

## 반복 컴포넌트 동일성 기하 검증

같은 이름의 반복 컴포넌트 인스턴스는 개별 overflow 검증 후 서로 상대 좌표를 비교한다.

검사 대상:

- `Organism/RecruitmentCard/Default`
- `Organism/RecruitmentCard/Compact`
- `PanelRow`
- 필터 옵션 칩 row

검증 방식:

- 각 반복 컴포넌트의 parent-relative child bounding box를 수집한다.
- 같은 역할의 child node는 x, y, width, height 차이를 비교한다.
- 텍스트 길이 때문에 width가 달라질 수 있는 요소는 제외하되, 시작 x, top y, row height, 버튼 right edge는 고정 기준으로 본다.

불합격 조건:

- 같은 역할의 액션 버튼 right edge 차이가 2px 이상이다.
- 같은 역할의 행 top offset 차이가 2px 이상이다.
- 같은 역할의 row height 차이가 2px 이상이다.
- 반복 컴포넌트의 내부 spacing이 카드마다 다르게 보인다.

## 리스트 하단 여백 검증

최종 화면에서 리스트 마지막 항목이 viewport 또는 부모 섹션 하단에 붙어 있으면 `불합격`이다.

필수 기준:

- `Page/Desktop Matching Board`에서 마지막 모집 카드 bottom과 viewport bottom 사이에 최소 24px 이상 여백이 있는가
- 마지막 모집 카드 bottom과 `Layout/ContentGrid` bottom 사이에 최소 16px 이상 여백이 있는가
- 카드가 많아 한 화면에 모두 넣기 어렵다면 리스트 스크롤, 클리핑, 더보기 구조가 명확한가
- 모든 카드를 억지로 viewport 안에 넣기 위해 카드 내부 padding이나 액션 행 정렬이 깨지지 않았는가

불합격 조건:

- 마지막 카드가 viewport 하단에 24px 미만으로 붙어 보인다.
- 카드 리스트가 하단 safe area 없이 끝난다.
- 하단 여백 확보를 위해 카드 내부 요소가 눌리거나 버튼 위치가 흔들린다.

## MVP 즉시 매칭 검증

- 예약제 UI가 없는가
- 미래 시간대 선택 UI가 없는가
- `오늘 가능`, `이번 주 가능`, `시간 조율`, `평일 21:00`, `주말 오후`, `예약` 같은 표현이 없는가
- 모집 카드가 지금 가능한 사람 중심으로 보이는가
- 필요 상태값은 `지금 가능`, `바로 1판`, `바로 2판`, `현재 플레이 중`처럼 즉시 가능 상태로 제한됐는가

## MVP 디스코드 필드 검증

- 필드 타이틀이 `디스코드`인가
- 선택지가 `대화 가능`, `듣기만 가능`, `디스코드 불가능` 3개뿐인가
- 모집 카드의 디스코드 배지도 위 3개 값 중 하나인가
- `Discord 가능`, `음성 필수`, `음성 선택`, `음성 없이 가능` 같은 이전 표현이 남아 있지 않은가

검증 방식:

- 반복 카드가 여러 개면 모든 카드를 각각 검증한다.
- 반복 리스트 행이 여러 개면 모든 행을 각각 검증한다.
- 같은 컴포넌트로 보여도 실제 화면에 놓인 인스턴스가 다르면 각각 검증한다.
- 축소 스크린샷에서 판단이 어려우면 해당 프레임 또는 노드를 확대 캡처한다.
- 판단하지 못한 요소가 하나라도 있으면 `불합격`이다.
