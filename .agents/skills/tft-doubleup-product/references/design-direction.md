# 디자인 방향

## 컨셉

전체 컨셉은 "TFT 정보 도구 + 랭크 듀오 로비"다. 화면은 신뢰감 있고, 촘촘하며, 빠르게 스캔할 수 있어야 한다.

참고 분위기:

- lolchess.gg 같은 정보 밀도
- 다크 대시보드
- TFT 전적/메타 사이트의 명확함
- 작은 배지, 태그, 리스트, 테이블 중심

lolchess.gg의 레이아웃, 로고, 고유 아이콘, Riot 자산, 세부 시각 표현을 직접 복제하지 않는다.

## 시각 톤

사용한다:

- 어두운 네이비 또는 차콜 배경
- 배경보다 살짝 밝은 패널
- 약한 대비의 테두리
- 티어 색상을 활용한 작은 배지
- 골드, 시안, 블루 포인트를 절제해서 사용
- 촘촘한 타이포그래피
- 테이블과 리스트 카드

피한다:

- 첫 화면을 큰 마케팅 히어로로 시작하는 구조
- 장식용 그라데이션 덩어리
- 보라/파랑 그라데이션만 반복되는 단조로운 팔레트
- 너무 귀여운 커뮤니티 게시판 느낌
- 정보 밀도가 낮은 과하게 큰 카드
- 카드 안에 또 카드를 넣는 구조

## 데스크톱 메인 화면

권장 구조:

```text
상단 네비게이션
로고 | 팀원찾기 | 모집글 | 랭킹 | 가이드 | 로그인

검색/필터 영역
Riot ID 또는 키워드 검색 | KR 고정 | 티어 | 목적 | 디스코드 | 지금 가능

콘텐츠 그리드
왼쪽: 필터 패널
가운데: 모집글 리스트
오른쪽: 최근 모집글 / 내 신청 상태 / 서비스 공지
```

첫 화면 안에서 실제 매칭 콘텐츠가 바로 보여야 한다.

MVP에서는 서버 선택 드롭다운을 만들지 않는다. 서버는 한국 서버(KR)로 고정 표시하고, 필터의 중심을 티어/목적/디스코드/플레이 스타일에 둔다. 플레이 시간 예약 필터는 만들지 않고, 지금 가능한 사람 중심으로 보여준다.
MVP에서는 `파트너 추천`, `랭킹`, `가이드`를 실제 기능으로 만들지 않는다. 네비게이션 항목은 둘 수 있지만 클릭 시 `준비중입니다.`를 보여준다. 메인 화면 안에도 추천 파트너, 랭킹, 가이드 콘텐츠 패널을 만들지 않는다.

## 핵심 컴포넌트

- 상단 네비게이션
- 검색창
- 필터 사이드바
- 목적/상태 세그먼트 컨트롤
- KR 고정 표시
- 티어 선택 필드
- 지금 가능 토글
- 모집 카드
- 티어 배지
- 디스코드 배지
- 플레이 스타일 태그
- 프로필 요약 카드
- 신청 버튼
- 준비중 모달 또는 준비중 상태
- 빈 상태
- 로딩 스켈레톤

## 모집 카드 구조

정보 순서:

1. KR 서버, 티어, 상태
2. 제목
3. 목적, 디스코드 대화 방식, 즉시 플레이 가능 상태
4. 플레이 스타일 태그
5. 작성자 미니 프로필과 신청 액션

카드 radius는 6-8px 정도로 둔다. 테두리는 은은하게 쓰고, 주요 액션은 잘 보이되 과하게 키우지 않는다.

## 모바일 화면

사용한다:

- 고정 검색 영역
- 필터 버튼과 바텀시트
- 단일 컬럼 모집 카드
- 상세 화면 하단 고정 액션

피한다:

- 모바일에서 항상 보이는 왼쪽 사이드바
- 핵심 리스트에 가로 스크롤이 필요한 테이블

## Figma 제작 메모

레이어 이름 예시:

- `Page/Desktop Matching Board`
- `Nav/Top`
- `Filters/Sidebar`
- `RecruitmentCard/Default`
- `Badge/Tier`
- `Badge/Availability`
- `Panel/RecentPosts`
- `Panel/MyRequests`
- `Modal/ComingSoon`

반복 구조는 오토레이아웃으로 만든다. 배지, 태그, 버튼, 카드는 여러 화면에서 재사용할 수 있도록 컴포넌트화한다.
