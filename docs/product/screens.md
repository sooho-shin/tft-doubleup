# 화면 명세

## 원칙

- 첫 화면은 랜딩페이지가 아니라 실제 팀원찾기 화면이다.
- MVP의 중심 화면은 데스크톱 매칭 보드다.
- 모바일은 같은 정보를 단일 컬럼과 바텀시트 필터로 재배치한다.
- `가이드`는 MVP에서 준비중 상태만 제공한다.
- `파트너 추천`과 `랭킹`은 MVP 상단 네비게이션에 노출하지 않는다.

## 화면 목록

### 더블업 팀원찾기 보드

목적:

- 사용자가 지금 모집 중인 더블업 파트너 방을 빠르게 스캔하고 신청하거나 직접 방을 만들게 한다.
- 이 화면은 방 리스트가 있는 메인 로비이며, 상세 기획은 `docs/product/lobby.md`를 따른다.

주요 영역:

- 상단 네비게이션
- 방 만들기 주요 액션
- 검색/필터 바
- 왼쪽 필터 패널
- 중앙 모집방 리스트
- 오른쪽 보조 패널

필수 상태:

- 기본 목록
- 필터 적용 목록
- 조건에 맞는 모집방 없음
- 로딩
- 준비중 모달

### 방 상세

목적:

- 사용자가 카드보다 자세한 설명, 방장 프로필, 신청 가능 여부를 확인한다.

주요 정보:

- 제목
- KR 서버
- 티어 범위
- 목적
- 디스코드
- 지금 가능 상태
- 플레이 스타일 태그
- 상세 설명
- 방장 프로필 요약
- 방장 본인 성별
- 신청 버튼

필수 상태:

- 신청 가능
- 이미 신청함
- 모집 마감
- 매칭 완료

### 방 만들기

목적:

- 사용자가 원하는 파트너 조건을 입력하고 즉시 모집 가능한 방을 만든다.

필수 입력:

- 방 제목
- 내 더블업 티어
- 원하는 티어 범위
- 목적
- 디스코드
- 플레이 스타일
- 지금 가능 상태
- 본인 성별
- 한 줄 설명

선택 입력:

- 원하는 파트너
- 상세 설명

규칙:

- 서버는 KR 고정으로 표시하고 사용자가 바꾸지 못한다.
- 예약 시간 입력은 제공하지 않는다.
- 미래 가능 시간 입력은 제공하지 않는다.
- 시간 조율 입력은 제공하지 않는다.
- 생성 완료 후 생성된 방 상세로 이동한다.
- 본인 성별 기본값은 `비공개`다.

필수 상태:

- 기본 입력
- 필수값 누락
- 생성 중
- 생성 실패
- 생성 완료 후 방 상세 이동

### 파트너 신청

목적:

- 사용자가 방장에게 짧은 신청 메시지를 보낸다.

필수 입력:

- 신청 메시지
- 본인 성별

보조 정보:

- 내 프로필 요약
- 상대 모집 조건 요약

규칙:

- 같은 방에 활성 신청은 하나만 가능하다.
- 신청 후 상태는 대기중으로 표시한다.
- 본인 성별 기본값은 `비공개`다.
- 성별은 신청자 본인 정보로만 표시하고 자동 매칭 조건으로 쓰지 않는다.

### 내 더블업 프로필

목적:

- 사용자가 반복 매칭에 쓸 프로필을 관리한다.

필수 입력:

- 표시 이름
- Riot ID
- 더블업 티어
- 본인 성별
- 디스코드
- 목적
- 플레이 스타일
- 지금 가능 상태
- 연락 채널
- 한 줄 소개

규칙:

- Riot ID는 MVP에서 직접 검증하지 않는다.
- 연락 채널은 신청이 수락된 사용자에게만 노출한다.

### 준비중 상태

대상:

- 가이드

표시 문구:

- `준비중입니다.`
- `MVP에서는 아직 제공하지 않는 기능입니다.`
- `지금은 모집방에서 바로 가능한 파트너를 찾아보세요.`

## 데스크톱 레이아웃

권장 구조:

```text
상단 네비게이션
로고 | 팀원찾기 | 모집방 | 가이드 | 내 프로필 | 로그인 | 방 만들기

검색/필터 바
키워드 검색 | KR 고정 | 티어 | 목적 | 디스코드 | 지금 가능

콘텐츠
왼쪽: 필터 패널
가운데: 모집방 리스트
오른쪽: 내 신청 상태 / 최근 모집방 / 서비스 공지
```

## 모바일 레이아웃

권장 구조:

```text
상단 바
더블업 팀원찾기 | 로그인

고정 검색 영역
키워드 검색 | 필터 버튼

콘텐츠
단일 컬럼 모집 카드

주요 액션
방 만들기

필터
바텀시트

상세
하단 고정 신청 액션
```

피해야 할 것:

- 모바일에서 항상 보이는 왼쪽 사이드바
- 모집방 리스트에 가로 스크롤이 필요한 테이블
- 카드 안에 또 다른 카드가 중첩되는 구조
