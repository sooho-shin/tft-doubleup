# MVP 범위

## 제품 약속

TFT 더블업 유저가 한국 서버(KR)에서 지금 바로 같이 플레이할 파트너를 빠르게 찾고, 티어/목적/디스코드/플레이 스타일이 맞지 않아 생기는 실패 확률을 줄인다.

## 핵심 사용자

- 더블업 랭크를 올리고 싶은 유저
- 지금 바로 큐를 돌릴 파트너를 찾는 유저
- 즐겜/빡겜 성향이 맞는 팀원을 찾는 유저
- 초보나 복귀 유저와 편하게 플레이할 파트너를 찾는 유저

## MVP 사용자 흐름

1. 사용자는 더블업 팀원찾기 보드에 진입한다.
2. 사용자는 KR 서버 고정 상태에서 모집방 목록을 본다.
3. 사용자는 티어, 목적, 디스코드, 플레이 스타일, 지금 가능 여부로 필터링한다.
4. 사용자는 방 카드에서 적합성을 빠르게 판단한다.
5. 사용자는 방 상세를 열어 설명과 방장 프로필을 확인한다.
6. 사용자는 신청 메시지를 보내거나 직접 방을 만든다.
7. 방을 만드는 사용자는 원하는 파트너 조건을 입력하고 방을 생성한다.
8. 방 생성이 완료되면 생성된 방 상세로 이동한다.
9. 방장은 신청을 수락하거나 거절한다.
10. 수락된 사용자는 연락 정보를 확인한다.

## 반드시 포함

- 모집방 목록
- 한국 서버(KR) 고정 표시
- 지금 가능 중심 목록
- 티어 범위 필터
- 목적 필터
- 디스코드 필터
- 플레이 스타일 필터
- 방 상세
- 방 만들기
- 사용자 프로필
- 신청/수락/거절 흐름
- 모집 상태: `모집중`, `매칭완료`, `마감`
- 준비중 상태: `가이드`

## MVP에서 제외

- 다중 서버 지원
- 예약 모집
- 미래 시간대 기반 모집
- 시간 조율 기능
- Riot ID 인증
- Riot API 기반 티어 동기화
- 실제 파트너 추천 점수
- 실제 파트너 추천 화면
- 실제 랭킹 화면
- 실제 가이드 콘텐츠
- 게임 후 매너 후기
- 서비스 내 채팅
- 범용 커뮤니티 피드
- 길드/팀 시스템
- 큰 마케팅 랜딩페이지

## 메인 화면 정보 구조

- 상단 네비게이션: `팀원찾기`, `모집방`, `가이드`, `내 프로필`, `로그인`
- 주요 액션: `방 만들기`
- 검색/필터 영역: 키워드, KR 고정, 티어, 목적, 디스코드, 지금 가능
- 왼쪽 패널: 상세 필터
- 중앙 영역: 모집방 리스트
- 오른쪽 패널: 내 신청 상태, 최근 모집방, 서비스 공지

## 방 카드 우선순위

1. KR 서버와 티어
2. 지금 가능 여부
3. 목적
4. 디스코드 대화 방식
5. 플레이 스타일 태그
6. 짧은 제목
7. 작성자 미니 프로필
8. 신청 버튼

예시:

```text
KR · Emerald II · 지금 가능
랭크 같이 올릴 파트너 구합니다
대화 가능 · 순방 위주 · 바로 2판 가능
#운영덱 #소통선호 #리롤가능
신청하기
```

## 방 만들기 흐름

방 만들기는 MVP 핵심 루프다. 목록과 상세만 있으면 사용자가 새 파트너 모집을 만들 수 없으므로, 사용자가 직접 조건을 걸고 방을 생성하는 화면을 반드시 제공한다.

흐름:

1. 사용자는 메인 보드 또는 상단 네비게이션에서 `방 만들기`를 누른다.
2. 사용자는 KR 고정 상태를 확인한다.
3. 사용자는 방 제목, 티어 범위, 목적, 디스코드, 플레이 스타일, 지금 가능 상태를 입력한다.
4. 사용자는 본인 성별 공개값, 원하는 파트너 조건, 한 줄 설명을 입력한다.
5. 사용자는 `방 만들기`를 눌러 방을 생성한다.
6. 생성 완료 후 생성된 방 상세로 이동한다.

필수 입력:

- 방 제목
- 내 더블업 티어
- 원하는 티어 범위
- 목적
- 디스코드
- 플레이 스타일
- 지금 가능 상태
- 본인 성별
- 한 줄 설명

선택 입력:

- 원하는 파트너
- 상세 설명

고정값:

- 서버: `KR`

MVP에서 제공하지 않는 입력:

- 예약 시간
- 오늘/이번 주 가능 여부
- 미래 시간대
- 시간 조율
- 다중 서버 선택
- Riot API 인증

## 신청 흐름

파트너 신청에서는 신청 메시지와 함께 신청자의 본인 성별 공개값을 입력할 수 있다.

필수 입력:

- 신청 메시지
- 본인 성별

성별 선택지:

- 남성
- 여성
- 비공개

규칙:

- 기본값은 `비공개`다.
- 성별은 사용자가 직접 선택한 본인 정보로만 표시한다.
- MVP에서는 성별 기반 필터나 추천 점수를 제공하지 않는다.
- 성별을 이유로 매칭 적합성을 자동 판단하지 않는다.

## 준비중 정책

MVP에서 `가이드`를 누르면 실제 화면으로 이동하지 않고 준비중 상태를 보여준다.

`파트너 추천`과 `랭킹`은 MVP 상단 네비게이션에 노출하지 않는다. 실제 기능과 진입점 모두 MVP 이후 확장으로 둔다.

사용 문구:

- `준비중입니다.`
- `MVP에서는 아직 제공하지 않는 기능입니다.`
- `지금은 모집방에서 바로 가능한 파트너를 찾아보세요.`
