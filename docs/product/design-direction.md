# 디자인 방향

## 컨셉

전체 컨셉은 `TFT 정보 도구 + 랭크 듀오 로비`다. 화면은 신뢰감 있고, 촘촘하며, 빠르게 스캔할 수 있어야 한다.

참고 분위기:

- lolchess.gg 같은 정보 밀도
- 다크 대시보드
- TFT 전적/메타 사이트의 명확함
- 작은 배지, 태그, 리스트, 테이블 중심

주의:

- lolchess.gg의 레이아웃, 로고, 고유 아이콘, Riot 자산, 세부 시각 표현을 직접 복제하지 않는다.
- Riot, TFT, lolchess의 브랜드 자산을 직접 사용하지 않는다.

## 시각 톤

사용한다:

- 어두운 네이비 또는 차콜 배경
- 배경보다 살짝 밝은 패널
- 약한 대비의 테두리
- 티어 색상을 활용한 작은 배지
- 골드, 시안, 블루 포인트를 절제해서 사용
- 촘촘한 타이포그래피
- 테이블과 리스트 카드

피한다:

- 첫 화면을 큰 마케팅 히어로로 시작하는 구조
- 장식용 그라데이션 덩어리
- 보라/파랑 그라데이션만 반복되는 단조로운 팔레트
- 너무 귀여운 커뮤니티 게시판 느낌
- 정보 밀도가 낮은 과하게 큰 카드
- 카드 안에 또 카드를 넣는 구조

## 색상 기준

- 배경: `#0E1118`
- 기본 패널: `#171B24`
- 강조 패널: `#202633`
- 테두리: `#2C3445`
- 기본 텍스트: `#F4F0E6`
- 보조 텍스트: `#A9B1C3`
- 약한 텍스트: `#717B91`
- 골드 포인트: `#D8B45C`
- 시안 포인트: `#36D1C4`
- 블루 포인트: `#4C8DFF`
- 위험: `#E25555`
- 성공: `#55C27A`

티어 색상은 전체 배경이 아니라 작은 배지나 포인트에만 쓴다.

## 핵심 컴포넌트

- 상단 네비게이션
- 검색창
- 필터 사이드바
- 목적/상태 세그먼트 컨트롤
- KR 고정 표시
- 티어 선택 필드
- 지금 가능 토글
- 모집 카드
- 티어 배지
- 디스코드 배지
- 플레이 스타일 태그
- 프로필 요약 카드
- 신청 버튼
- 준비중 모달 또는 준비중 상태
- 빈 상태
- 로딩 스켈레톤

## 모집 카드 구조

정보 순서:

1. KR 서버, 티어, 상태
2. 제목
3. 목적, 디스코드 대화 방식, 즉시 플레이 가능 상태
4. 플레이 스타일 태그
5. 작성자 미니 프로필과 신청 액션

시각 규칙:

- 카드 radius는 6-8px 정도로 둔다.
- 테두리는 은은하게 쓴다.
- 주요 액션은 잘 보이되 과하게 키우지 않는다.
- 카드 내부에 별도 카드를 중첩하지 않는다.

## Figma 제작 메모

레이어 이름 예시:

- `Page/Desktop Matching Board`
- `Nav/Top`
- `Filters/Sidebar`
- `RecruitmentCard/Default`
- `Badge/Tier`
- `Badge/Availability`
- `Panel/RecentPosts`
- `Panel/MyRequests`
- `Modal/ComingSoon`

반복 구조는 오토레이아웃으로 만든다. 배지, 태그, 버튼, 카드는 여러 화면에서 재사용할 수 있도록 컴포넌트화한다.
