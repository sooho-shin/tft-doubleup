# 로비 기획

## 정의

로비는 사용자가 서비스에 들어왔을 때 처음 만나는 메인페이지다. 이 화면은 마케팅 랜딩페이지가 아니라, 방 리스트를 중심으로 지금 같이 할 더블업 파트너를 찾는 작업 화면이다.

사용자에게는 `더블업 팀원찾기` 또는 `모집방`으로 노출하고, 기획상으로는 로비라고 부른다.

## 로비의 핵심 목표

로비의 1순위 성공 행동은 사용자가 `신청할 방이 있는지`와 `직접 방을 만들어야 하는지`를 빠르게 판단하는 것이다.

사용자는 로비에서 다음 순서로 판단한다.

1. 지금 바로 신청할 만한 방이 있는지 본다.
2. 내 티어와 목적에 맞는 방인지 확인한다.
3. 디스코드와 플레이 스타일이 크게 맞지 않는지 훑는다.
4. 맞는 방이 있으면 `신청하기`를 누른다.
5. 맞는 방이 없으면 `방 만들기`로 전환한다.

따라서 로비는 단순 목록이 아니라 `방 찾기`와 `방 만들기` 사이의 빠른 선택을 돕는 화면이어야 한다.

## 판단 기준 우선순위

로비에서 가장 먼저 보여줘야 하는 기준은 다음 3개다.

1. 지금 가능 여부
2. 티어 범위
3. 목적

보조 판단 기준은 다음 2개다.

- 디스코드
- 플레이 스타일

`현재 대기 시간`, `마감 임박`, `예약 시간`, `시간 조율`은 MVP 도메인에 없으므로 로비 기획에 넣지 않는다.

## 데스크톱 정보 구조

```text
상단 네비게이션
로고 | 팀원찾기 | 모집방 | 가이드 | 내 프로필 | 로그인 | 방 만들기

로비 상단
더블업 팀원찾기 | KR 고정 | 모집중 N개 | 지금 가능 N개 | 방 만들기

검색/빠른 필터
키워드 검색 | 티어 | 목적 | 디스코드 | 지금 가능 | 필터 초기화

본문
왼쪽: 상세 필터
가운데: 모집방 리스트
오른쪽: 내 신청 상태 / 최근 모집방 / 서비스 공지
```

로비 상단은 큰 히어로가 아니라 현재 상태를 빠르게 알려주는 얇은 요약 영역으로 둔다. `방 만들기`는 상단 네비게이션과 로비 상단 중 최소 한 곳에서 항상 보이게 한다.

## 모바일 정보 구조

```text
상단 바
더블업 팀원찾기 | 로그인

로비 요약
KR 고정 | 모집중 N개 | 지금 가능 N개

검색/필터
키워드 검색 | 필터 버튼

본문
단일 컬럼 모집방 카드

주요 액션
방 만들기

필터
바텀시트
```

모바일에서는 왼쪽 필터 패널을 노출하지 않는다. 필터는 바텀시트로 열고, `방 만들기`는 리스트를 보다가도 접근 가능한 주요 액션으로 둔다.

## 모집방 카드 구조

카드는 다음 순서로 읽히게 만든다.

```text
KR · Emerald II-Diamond IV · 지금 가능
랭크 상승 파트너 구해요
대화 가능 · 순방 위주 · 바로 2판 가능
#운영덱 #소통선호 #리롤가능
방장: 레벨업타이밍 · Emerald II
신청하기
```

필수 요소:

- KR 서버 고정 표시
- 티어 범위
- 지금 가능 상태
- 방 제목
- 목적
- 디스코드
- 플레이 스타일 태그
- 방장 미니 프로필
- 신청 버튼

카드는 정보 밀도를 유지하되, 카드 안에 또 다른 카드를 중첩하지 않는다.

## 필터 전략

기본 목록은 `모집중` 방을 우선 보여준다. `지금 가능` 방은 기본 정렬에서 위로 올리되, 사용자가 `지금 가능` 필터를 켜면 지금 가능한 방만 남긴다.

필터 우선순위:

1. 지금 가능
2. 티어
3. 목적
4. 디스코드
5. 플레이 스타일

성별 필터는 MVP에서 제공하지 않는다.

## 방 만들기 유도 상태

다음 상황에서는 `방 만들기`를 강하게 유도한다.

- 검색 결과 0개
- 지금 가능 방 없음
- 내 티어 범위와 맞는 방 없음

표시 문구:

```text
조건에 맞는 모집방이 없습니다.
원하는 조건으로 방을 만들어 파트너를 기다려보세요.
```

주요 액션:

- `방 만들기`
- `필터 초기화`

다음 상황에서는 보조 유도로 처리한다.

- 목적이 맞는 방 없음
- 디스코드나 플레이 스타일이 안 맞음

표시 문구:

```text
필터를 줄이거나 직접 방을 만들어보세요.
```

## 필수 상태

로비는 다음 상태를 갖는다.

- 기본 목록
- 필터 적용 목록
- 조건에 맞는 모집방 없음
- 지금 가능 방 없음
- 내 티어 범위와 맞는 방 없음
- 로딩
- 준비중 모달

로딩 상태에서는 모집방 카드 형태의 스켈레톤을 보여준다. 빈 상태에서는 이유와 다음 행동을 같이 보여준다.

## 설계 원칙

- 첫 화면을 큰 홍보 문구로 시작하지 않는다.
- 로비의 중심은 방 리스트다.
- 사용자가 5초 안에 `신청하기` 또는 `방 만들기` 중 하나를 고를 수 있어야 한다.
- 티어 색상은 작은 배지나 포인트에만 쓴다.
- `파트너 추천`, `랭킹`, 예약 모집, 시간 조율은 MVP 로비에 넣지 않는다.
- lolchess.gg는 정보 밀도와 다크 도구 느낌만 참고하고 레이아웃이나 고유 표현은 복제하지 않는다.
